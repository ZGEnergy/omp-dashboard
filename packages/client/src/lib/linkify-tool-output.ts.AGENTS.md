# linkify-tool-output.ts — index

Pure tokeniser. Exports `tokenize(text): Token[]`, `MAX_LINKS=5000`. Detects URLs, `path:line[:col]`, bare path with generic extension `EXT=[A-Za-z][A-Za-z0-9]{0,15}` (alpha-first, ≤16 chars). Single linear pass. Precedence URL > path-with-line > path-with-ext. Coverage invariant (non-overflow): `tokens.map(t=>t.text).join("") === input`. Overflow caps 5000 link tokens, appends `\n+N more links suppressed` text token. Strips URL trailing punctuation `[.,;:!?)\]}>'"]`. See change: linkify-tool-output. Detects absolute POSIX paths, `file://`/`file:///` URIs (percent-decoded), Windows drive paths. `file` Token gains `absolute?: boolean`. Absolute branches fold trailing `:line[:col]`; drive colon never parsed as line separator. See change: unify-file-link-openability. Generic extension replaces fixed `EXTS`/`EXT_GROUP` allowlist; any ext linkifies when path structure present. Bug A: `.json` no longer truncated to `.js` (no enumerated alternation, no prefix collision). Bug B: relative grammar admits leading-dot dirs (`.pi`, `.github`) as leading segment after separator, interior segments via `RDIR=\.?[\w][\w.-]*`; bare first segment needs separator so `1.2.3`/`Node.js` stay non-pathy. Bug C: relative prefix `(?:\.{1,2}\/)+` (one-or-more `../`); `../../foo.ts` matches whole, marked NOT absolute (file_posix never re-captures interior `/...` tail as bogus absolute). Unchanged: `Token` shape, `tokenize` signature, `splitLineCol`, absolute-marking. See change: linkify-any-text-extension.
