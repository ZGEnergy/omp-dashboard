# chat-virtual-rows.ts — index

Pure helpers for the windowed (TanStack Virtual) transcript. Exports `isBurst`/`isGroup` guards, `virtualRowKey(item,index)` (CR-3: `burst.id` \| first-member key \| `group-<index>` \| messageKey = `nonce ?? entryId ?? msg.id` — position-independent so older-page prepends cannot re-point scroll anchors), `estimateVirtualRowSize(item, textChars?)` (CONTENT-AWARE pre-measure: `base + ceil(textChars/80)*20` clamped 8000px + per-kind image reserve 300 user/512 toolResult; burst 220/group 64 constants), `computeRowTextChars(item)` (aggregate rendered text length, walked once in ChatView useMemo), `buildTurnToFirstRowIndex(rows)` (CR-4: `turnIndex`→first display-row index over the FILTERED rows, backs `scrollToTurn`), `rangeToRowIndexSpan(range, container, rowCount)` (DOM Range → inclusive `{min,max}` display-row span; walks endpoints to nearest `[data-index]`, clamps non-virtual endpoints via `compareDocumentPosition`, normalizes reversed/same-row, null when no virtual row touched), `extendRangeWithSelection(base, span, cap, count)` (union selection span into `defaultRangeExtractor` output; returns `base` unchanged past `cap` so caller actively clears — no full mount). See change: preserve-chat-selection-during-churn. See change: virtualize-chat-transcript-tanstack. See change: fix-chat-scroll-to-top-estimate-drift — static per-role estimate under-shot large top rows 10-50x so scroll-up never converged; content-aware estimate shrinks the delta that drives TanStack's built-in above-viewport correction. `rowSeq(item)` (exported): best-effort seq per row, first underlying member for burst/group, `toSeq` for an evicted-burst marker. `lowestVisibleSeq(rows, startIndex, endIndex)`: lowest seq in an inclusive virtualizer range, feeds `App.tsx` viewport-floor pin (Task 1.7, change: bounded-hot-transcript-state). `DisplayRow` type = `BurstItem | EvictedToolBurst`. `isEvictedBurst(item)`: structural guard, no `type`/`role` key, has `fromSeq`. `interleaveEvictedBursts(rows, bursts)`: inserts each `EvictedToolBurst` marker at its seq position among `rows` (ascending, NOT unconditionally at top) — fixes toolFloorSeq>chatFloorSeq double-render defect (marker + still-rendered tool row).
