# replay-coordinator.ts

Owns ordered browser event replay delivery per socket/session. Cold, delta, and older finite replay frames are admitted incrementally and serialized behind the asynchronous drain so finite history does not prefill queue caps; suppressed or unbounded live traffic remains subject to event/byte overflow (1013). Truncated retained heads retain generation-cached persisted source pages: cold tails advertise persisted older history and boundary older requests select canonical persisted pages without replacing the bounded store. Cold/older windows select against the full wire budget across multiple frames (per-event prep capped at frame size); over-budget plans trim whole batches from the oldest end so delivery always stays a contiguous suffix anchored at the newest event, and inline asset registrations stage until the final window survives trimming. Request terminal frames and live replay barriers remain ordered, and `REPLAY_SEND_BACKPRESSURE` requeues the current item without treating temporary gateway pressure as overflow.

Compacts superseded assistant stream snapshots before byte selection: inert updates preserve sequence continuity; terminal `message_end` restores completed content; newest unfinished text and thinking deltas stay replayable.
