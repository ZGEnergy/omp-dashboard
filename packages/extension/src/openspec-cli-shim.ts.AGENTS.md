# openspec-cli-shim.ts — index

Provision bare `openspec` inside a pi session so the generated `openspec-*` skills' bare-name CLI calls resolve (the sole in-session lever is `process.env.PATH`, which `getShellEnv()` reads live per bash child). At bridge `session_start` (init AND `/reload`): resolve the pinned `@fission-ai/openspec` bin (`@fission-ai/openspec` exports only `.`, so `require.resolve` the main entry then walk up to the package root and join `bin/openspec.js` — the `/bin/*` and `/package.json` subpaths throw `ERR_PACKAGE_PATH_NOT_EXPORTED`); ensure a `0700` shim dir `~/.pi/dashboard/openspec-shim`; write an EXTENSIONLESS `#!/bin/sh` shim `exec "<process.execPath>" "<bin.js>" "$@"` (absolute node → survives a stripped PATH; extensionless → resolves under Git Bash `bash.exe -c`, which ignores PATHEXT so a `.cmd` would not) atomically (temp+rename) and RE-POINTED every init; idempotently prepend the canonical (realpath) shim dir to `env.PATH` (compare realpath forms, prepend-only, non-destructive). Fail-soft: on resolve/write error, log + invoke `onMissingTool` (bridge sends a `missingTool`-style `bash_output` for `openspec`), never throw.

Exports `provisionOpenspecCli(deps)` (orchestrator; `deps` seams `resolveBin`/`baseDir`/`execPath`/`env`/`log`/`onMissingTool` for hermetic tests), `resolveOpenspecBin`, `ensureShimDir`, `writeShim`, `prependDirToPath`. Version single-sourced at 1.6.0 (see `openspec-cli-version-single-source`). Wired in `bridge.ts` `session_start`; covered by `__tests__/openspec-cli-shim.test.ts` (T-E1..E4, X1, X2) + qa `12-openspec-shim.ps1` (T-C1).
