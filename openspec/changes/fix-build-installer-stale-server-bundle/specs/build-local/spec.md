## MODIFIED Requirements

### Requirement: build:local npm script
The `packages/electron/package.json` SHALL declare an npm script `build:local` that produces a fully offline-capable Electron app for the local host platform with a single command. The script SHALL default `BUNDLE_OFFLINE_PACKAGES=1` so the resulting installer ships with the offline npm cacache for `pi-coding-agent`, `openspec`, and `tsx`.

The build pipeline SHALL detect when the cached `resources/server/` bundle is stale relative to its source inputs and re-run `bundle-server.mjs` automatically. Cache invalidation by directory existence alone is insufficient — source edits to any tracked workspace MUST trigger a re-bundle on the next build invocation without requiring `clean:resources`.

#### Scenario: Single-command local build
- **WHEN** a developer runs `npm run build:local` in `packages/electron/`
- **THEN** the build SHALL produce an installer in `out/make/` for the host platform
- **AND** the installer SHALL contain the bundled offline cacache at `resources/offline-packages/`
- **AND** the installer SHALL contain the bundled Node runtime at `resources/node/`
- **AND** the installer SHALL contain the bundled server at `resources/server/`

#### Scenario: User env overrides preserved
- **WHEN** a developer runs `BUNDLE_OFFLINE_PACKAGES=0 npm run build:local`
- **THEN** the offline cacache SHALL NOT be bundled
- **AND** the resulting installer SHALL behave as today's `npm run make` (online-only first launch)

#### Scenario: Source edit triggers re-bundle
- **WHEN** a developer edits any file under `packages/{server,shared,client,extension,dashboard-plugin-runtime}/src/` (matching tracked extensions `*.ts`, `*.tsx`, `*.js`, `*.mjs`, `*.cjs`, `*.json`, `*.html`, `*.css`, `*.svg`) **AFTER** a previous `build:local` succeeded
- **AND** the developer runs `npm run build:local` again without touching `resources/server/`
- **THEN** `bundle-server.mjs` SHALL re-run
- **AND** the resulting installer SHALL contain the edited source under `resources/server/packages/<short>/src/`

#### Scenario: Bundler script edit triggers re-bundle
- **WHEN** a developer edits `packages/electron/scripts/bundle-server.mjs` **AFTER** a previous `build:local` succeeded
- **AND** runs `npm run build:local` again without touching any other file
- **THEN** `bundle-server.mjs` SHALL re-run
- **AND** the bundle stamp's `bundlerMtime` SHALL be updated to match the new script mtime

#### Scenario: Unchanged source skips bundler (cache hit)
- **WHEN** a developer runs `npm run build:local` twice in a row with no intervening source edits
- **THEN** the second invocation SHALL emit a log line matching `✓ Bundled server cache hit` and SHALL NOT invoke `bundle-server.mjs`
- **AND** the build SHALL still produce a valid installer
- **AND** the cache-hit decision SHALL complete in under 100 ms on warm filesystem cache

#### Scenario: Stamp absent forces rebuild
- **WHEN** `resources/server/` exists but the bundle-stamp file is absent (e.g. produced by a build before this gate shipped)
- **AND** the developer runs `npm run build:local`
- **THEN** `bundle-server.mjs` SHALL re-run
- **AND** the log line SHALL include `reason=stamp-missing`

#### Scenario: Cache decision is auditable in CI logs
- **WHEN** `build-installer.sh` runs in any environment
- **THEN** exactly one log line SHALL be emitted reporting the cache decision
- **AND** the line SHALL match either `^↻ Bundled server stale \(reason=<token>` (re-bundling) or `^✓ Bundled server cache hit` (skipping)
- **AND** the stale-reason token SHALL be one of `stamp-missing`, `source-newer`, `bundler-newer`

### Requirement: clean:resources script
The `packages/electron/package.json` SHALL declare an npm script `clean:resources` that removes the build outputs `resources/server`, `resources/node`, `resources/offline-packages`, and `resources/bundled-extensions`. This SHALL remain the canonical escape hatch when the auto-staleness gate fails (clock skew, fs without mtime support, manual cache surgery).

#### Scenario: Clean then rebuild
- **WHEN** a developer runs `npm run clean:resources && npm run build:local`
- **THEN** all four resource directories SHALL be removed and rebuilt fresh
- **AND** the bundle-stamp file SHALL be regenerated by the rebuilt `bundle-server.mjs` invocation

## ADDED Requirements

### Requirement: Bundle-stamp sentinel
After every successful `bundle-server.mjs` invocation, `build-installer.sh` SHALL write a JSON sentinel describing the bundle's source inputs at build time. The sentinel is the authoritative reference for subsequent staleness checks.

#### Scenario: Stamp written after bundle success
- **WHEN** `bundle-server.mjs` exits 0 inside `build-installer.sh`
- **THEN** a JSON file SHALL be written containing at minimum the fields `builtAt` (ISO 8601 UTC string), `srcMtime` (Unix epoch seconds of the newest tracked source file at build time), and `bundlerMtime` (Unix epoch seconds of `bundle-server.mjs` at build time)
- **AND** the file SHALL be valid JSON parseable by `JSON.parse`

#### Scenario: Stamp NOT written on bundler failure
- **WHEN** `bundle-server.mjs` exits non-zero
- **THEN** the stamp file SHALL NOT be created or updated
- **AND** the next `build:local` invocation SHALL still see the stamp from the previous successful build (if any), preventing a partial bundle from being mistaken for fresh

#### Scenario: Stamp not shipped to end users
- **WHEN** the Electron app is packaged
- **THEN** the bundle-stamp file SHALL NOT appear inside the shipped `.app` / `.dmg` / `.deb` / `.AppImage` / `.exe` payload
