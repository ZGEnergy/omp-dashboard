- [ ] 1.2 Define `SECTION_OF: Record<string, DoctorSection>` mapping every canonical check name to its section per the design taxonomy (runtime / pi-tooling / server / setup / diagnostics)

- [ ] 1.4 Implement `stripAnsi(input)` pure helper in `doctor-core.ts` (regex-based, no dependency); strips standard CSI / OSC escape sequences. Applied to every stderr tail before storage in `detail`
- [ ] 1.5 Implement `safeExec(cmd, opts)` in `doctor-core.ts` — wraps `execSync` with a configurable timeout (default 5000 ms; callers pass `timeoutMs: 15000` for cold-start probes — bundled-Node version probe and server-launch sanity test), `windowsHide: true`, and classifies outcomes into `{ ok: true, stdout } | { ok: false, kind: ExecFailureKind, message, detail, exitCode?, stderrTail? }` (stderr tail run through `stripAnsi`, then capped at 500 chars). Timeout failure suggestion text SHALL include the actual deadline ("did not respond within 5s" vs. "within 15s")
- [ ] 1.6 Implement `safeCheck(name, fn)` per-check fault-isolation wrapper — catches any throw / rejection from `fn` and returns an `error` row with non-empty `message` / `detail` / `suggestion`, never propagates
- [ ] 1.7 Implement `assumedMandatory(label, fn, deps)` — runs `fn`; on throw, appends a single timestamped JSON line to `<deps.managedDir>/doctor.log` (path derived from injected `MANAGED_DIR` so server / Electron / env-overrides all converge); BEFORE appending, performs ring rotation if the file exceeds 1 MB (rename `doctor.log` → `doctor.log.1`, replacing any existing `.1`); rotation + append both wrapped in `try/catch` and silently drop on failure; returns a `diagnostics`-section `error` row labelled `"Doctor internal: <label>"` with the captured error in `detail`
- [ ] 1.8 Implement `formatDoctorReportMarkdown(report)` — one Markdown table per non-empty section in the fixed order, summary header line, "Remediation" bullet list of `name → suggestion` for non-ok rows only; `detail` text in table cells SHALL be wrapped in fenced ` ```text ` blocks with HTML line-break encoding (`<br>`) so embedded pipe characters cannot break the table column count
- [ ] 1.9 Implement `runSharedChecks(deps)` covering the non-Electron checks (System Node.js, pi CLI, openspec CLI, tsx, Dashboard server HTTP probe, Server log presence, API key file presence, Managed install) — every external invocation routed through `safeExec`, every check wrapped in `safeCheck`, every "assumed safe" filesystem read wrapped in `assumedMandatory`; `deps` shape includes `managedDir: string` so the doctor log path follows env overrides / Electron's `MANAGED_DIR`

## 3. Phase A3 — Electron Doctor window

- [ ] 3.1 Create `packages/electron/src/lib/doctor-bridge-contract.ts` defining a `DoctorBridge` TypeScript interface (`run() / openLog() / openDoctorLog() / runSetup() / copy(text) / openManagedDir()`); this single interface is imported by BOTH the preload and the renderer entry, so a renamed channel breaks the type-check rather than silently breaking buttons (risk #4 mitigation)
- [ ] 3.2 Create `packages/electron/src/preload/doctor-preload.ts` implementing `DoctorBridge` via `contextBridge.exposeInMainWorld("electron", { doctor: <impl> })`
- [ ] 3.3 Create `packages/electron/src/renderer/doctor.html` — hand-rolled HTML+CSS reusing wizard.html tokens; renders one `<table>` per section with status pill, name, message, path (truncated, hover-full), and inline `.suggestion` row beneath non-ok rows; toolbar buttons `[Re-run]`, `[Copy as Markdown]`, `[Copy as Plain]`, `[Open server log]`, `[Open doctor log]`, `[Run setup wizard]`
- [ ] 3.4 Wrap the renderer's `doctor:run` handler in `try/catch`: on rejection or invalid shape, render a single fallback row ("Doctor failed to produce a report") with the captured error and keep `[Re-run]` + `[Open doctor log]` toolbar buttons enabled. While a run is in flight, the renderer disables the `[Re-run]` button and shows a "Running…" label / spinner (decision 10). When `[Open doctor log]` resolves with `{ exists: false }`, the renderer shows an inline status message "No doctor log yet — the doctor has not encountered internal failures." instead of opening anything (decision 4)
- [ ] 3.5 Create `packages/electron/src/lib/doctor-window.ts` — `openDoctorWindow()` factory: creates a single `BrowserWindow` (1000×720, resizable, title "PI Dashboard Doctor"); reuses the existing window if already open (focus instead of recreate); wires `window.on("closed", () => doctorWindow = null)` to prevent the destroyed-window leak (risk #3 mitigation); registers `ipcMain.handle("doctor:run", ...)`, `doctor:open-log`, `doctor:open-doctor-log`, `doctor:run-setup`, `doctor:copy`, `doctor:open-managed-dir` — every handler wraps its body in `try/catch` and rejects with a structured `{ kind, message, detail }` object on failure. The `doctor:run` handler short-circuits concurrent invocations: if a previous call is still pending, the second call awaits the first's result and returns the same report (re-run race serialization, decision 10). The `doctor:open-doctor-log` handler resolves with `{ exists: false }` when the log file is absent so the renderer can show the "No doctor log yet" toast without creating an empty file (decision 4); `doctor:run-setup` resets the wizard's persisted state to step 1 before opening the wizard window (decision 1)
- [ ] 3.6 Update `packages/electron/src/lib/app-menu.ts` — replace the `dialog.showMessageBox` Doctor handler with `openDoctorWindow()`; keep the menu entry text & accelerator
- [ ] 3.7 Add `[Run Doctor]` link in the wizard renderer (`packages/electron/src/lib/wizard-window.ts` / `wizard.html` footer) wired through a new preload IPC `wizard:open-doctor`
