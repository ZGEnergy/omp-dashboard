F · How subagents consume it

Spawn a subagent bound to a custom role

# Agent tool call — model field is a @role ref
Agent(
  subagent_type: "doubt-driven-review",
  model: "@doubt-verifier-x",   // resolves via role:resolve-model
  prompt: "stress-test this migration…"
)
resolution path
@doubt-verifier-x
   └▶ role:resolve-model  (role-manager.ts)
        └▶ providers.json#roles["doubt-verifier-x"]
             └▶ "anthropic/claude-opus-4"  ✓ spawned

If the role has no model, probe.reason = "not configured yet" → the harness surfaces an actionable spawn error instead of a silent fail. Unchanged from today.
