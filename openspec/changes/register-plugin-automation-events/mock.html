Naming — ambiguity killed

event = inbound TRIGGER (cron, git push, openspec) — what fires it.
action = outbound REGISTERED capability (core.prompt, flows.run) — what it does.
One word per direction. No collision in YAML, docs, or UI.

Namespacing — no clutter, no clash

id = "<source>.<verb>"
  flows.run ≠ git.run ≠ bare run
  source owns its group + ≤12 actions
  available(cwd) ⇒ whole group hidden

Clutter controls (in order)

1. available(cwd) — uninstalled / N/A sources don't render.
2. collapse by source — one accordion row per plugin.
3. search box — auto-shows past ~8 actions; type flow → jump.
4. cap ≤12/plugin — registry warns + truncates beyond.
