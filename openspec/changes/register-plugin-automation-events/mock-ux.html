Naming — validated

trigger (the inbound event) — cron / git / openspec. What fires it.
action — registered, namespaced capability. What it does.
Keep "event" in code/protocol; surface trigger in UI so one user-facing word maps to one concept — NN/g Consistency & Standards (#4), "Avoid confusing category names".

Scale controls (in order)

1. available(cwd) — un-configured plugins stay visible but disabled with a reason, not hidden (system status, recognition).
2. group by plugin — one accordion per source (Law of Proximity; chunking, Miller's Law).
3. search/filter — combobox over the grouped listbox; type to jump (Baymard long-list search).
4. cap ≤12/plugin — registry truncates + warns.

Selector pattern

Combobox + grouped listbox per WAI-ARIA APG. Search filters; ↑/↓ moves the active option, Enter selects, the same row count drives aria-live — so screen-reader and sighted users perceive list size identically.
